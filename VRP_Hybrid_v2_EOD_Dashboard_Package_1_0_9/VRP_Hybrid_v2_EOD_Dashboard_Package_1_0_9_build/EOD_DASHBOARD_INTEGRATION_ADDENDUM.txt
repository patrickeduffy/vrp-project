VRP Hybrid v2 — EOD Dashboard Production Integration Addendum
==============================================================
Release date: 2026-07-11
Model lock: vrp_corsi_intraday_hybrid_v2
Dashboard pipeline: vrp_hybrid_v2_eod

Purpose
-------
This addendum updates the production-integration boundary described in the Hybrid v2 model-lock documents.
The model methodology is unchanged. A consolidated completed-EOD production runner and Streamlit dashboard
now exist as a separate operational layer around the locked model.

New production integration
--------------------------
- Standalone orchestrator: notebooks\vrp_hybrid_v2_eod_pipeline.py
- Locked publisher: notebooks\vrp_hybrid_v2_signal_publish.py
- Semantic health gate: notebooks\vrp_hybrid_v2_health_check.py
- Accepted long-warmup RSI updater: notebooks\vrp_hybrid_v2_wilder_rsi_update.py
  - Release 1.0.1 extends the stored accepted Wilder state using canonical SPY EOD closes.
  - It does not call the ThetaData historical stock endpoint.
- Streamlit app: notebooks\streamlit_vrp_hybrid_v2_eod.py
- Launcher: launch_vrp_hybrid_v2_streamlit.bat

The orchestrator backfills missing and interior dates through the latest completed XNYS session, rebuilds
impacted derived history, reconstructs the exact locked intraday Ridge forecast, recalculates prior-only z-scores,
applies the Hybrid v2 thresholds/sizes/selector, and publishes dashboard outputs only after hard validations pass.

Unchanged model boundaries
--------------------------
- No intraday trading signal.
- No order placement.
- No automatic whole-contract rounding or live option-leg selection.
- No portfolio overlap cap, stress overlay, or hedge logic.
- A displayed TRADE remains a completed-EOD model decision requiring separate execution and portfolio approval.

Change control
--------------
The dashboard reads the versioned production configuration. It does not redefine the forecast, thresholds,
sizing, or selector. Any change to those components requires a new model-lock version.

Operational patch 1.0.1 — RSI subscription compatibility
---------------------------------------------------------
The initial operational updater incorrectly attempted a full 1993-present SPY history request on each
first run. ThetaData STANDARD subscriptions reject that stock endpoint with HTTP 403. The locked RSI
methodology does not require a new historical pull because the accepted canonical RSI parquet stores
the recursive Wilder average-gain and average-loss state. The patched updater preserves the accepted
history, repairs any interior or recent gaps forward from the last valid state, and validates exact
formula version, canonical closes, recursive fields, and target-date coverage before publishing.
